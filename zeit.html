Introduction

Brief

Create a responsive e-commerce website for the fictional company Zeit. Zeit is a subsidiary of Virgin that recently gained approval to sell time-travel experiences. Zeit will be selling trips to a total of 289 destinations around the world, from prehistoric times up through today.

Goals

  1. Design an effective logo for Zeit to create strong branding.
  2. Create a responsive ecommerce website that is easy to use.

Overview

Role: sole end-to-end UX/UI designer

Timeline: ~8 weeks

Design Process

Research

Research Goals

We want to...

  1. identify major demographic and psychometric characteristics of potential users
  2. understand how users choose destinations when using travel sites
  3. identify user attitudes regarding features and design of existing travel sites

in order to...

understand the user base for this brand-new product and create an enticing, enjoyable, user-friendly site for booking trips.

Competitive Analysis

My analysis of competitor travel sites showed that a strong travel site should have:

  1. trips categorized by destination and theme/interest
  2. powerful search and filter abilities
  3. background and context information for trips
  4. an easy booking experience
  5. a good user flow between pages
  6. an uncluttered visual design and layout with strong branding

User Interviews

I created an interview script based on the project brief, research goals, and competitive analysis. The first round of interviews involved 3 participants; although good data was collected from these interviews, I found that the first interview script was not soliciting enough information about how users select particular destinations on travel sites. A second round of interviews with 2 additional participants, conducted with an updated script, addressed this issue.

The interview revealed the users’ main motivations and pain points when using travel sites:

Motivations

  • Ease of use of site
  • Reliable information about prices
  • Strong price comparison tools
  • Good filter options

Pain Points

  • Slow loading times
  • Difficulty changing search options
  • Lack of accurate information
  • Lack of clear confirmation screens

Persona

The persona was created as a composite representative of the demographics and psychometrics of the users I interviewed. Like the interview participants, Emma appreciates travel sites that provide clear information about their trips and a easy booking process.

Empathy Map

Considering Emma’s environment. thoughts and feelings, pains, and gains provided valuable insights for developing the information architecture, task flows, and interaction designs for the site—particularly the site hierarchy and extensive filtering options.

Storyboard

Imagining a scenario in which Emma would encounter and use the site helped me understand potential problem areas and begin mapping out task and user flows based on how Emma would interact with different parts of the site.

Card Sorting

To understand how to categorize the trips Zeit offers in ways that would match users’ mental maps, I conducted an open card sorting activity using OptimalWorkshop. I created 30 trips cards spanning the wide range of trips that Zeit offers and asked 3 participants to sort them into categories.

The 3 participants grouped the trips by continent and themes. Because there was little overlap among the categories the original participants created, I decided to conduct the card sort activity with 2 additional participants to gather more data. The 2 additional participants sorted the cards into categories based on historical era and themes.

zoom_in

Site Map

Based on the results of the card sorting, I decided to categorize the trips by into geographic, historical, and thematic categories based on the categories created by the card sorting participants, with some modifications for clarity and consistency.

Task and User Flows

Based on the project brief, persona, and site map, I created task and user flows for Zeit to better understand how users would interact with the site. The task flow explores the purchasing of a trip, while the user flow explores searching and saving a trip for later purchase. Two other user flows, not depicted, demonstrate the processes for retrieving and purchasing a saved trip and for reviewing a completed trip.

Conclusion

Summary

Completing my first UX project was challenging, but rewarding. I learned that my current strengths are research, analysis, problem definition and solving, and wireframing, while I need to spend more time working on ideating through sketching and visual design. If I had more time with this project, I would consider lighter color schemes—although the current color palette matches the brand inspiration and overall desired feel for the site, it is a little dark and heavy.

My favorite part of the project was thinking of creative ways for users to interact with the homepage to browse trips. I wanted the site to inspire the same kind of wonder and awe in the user as the idea of time travel. I think the creative interactive elements of the homepage—the timeline and the map—help achieve this goal.

Next Steps

Were this project to continue, there are several steps I would take to improve the user experience and design.

  1. Add the new wireframes I created after usability testing to the prototype, conduct a new round of usability tests, iterate as needed, and incorporate the new design pattern into the responsive UI designs.
  2. Build out more of the designs for the mobile site for additional practice in creating mobile design patterns and mobile UI designs.
